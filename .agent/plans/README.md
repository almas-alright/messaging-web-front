# Plans

Place dated/named frontend execution plans here.

Recommended naming:

```text
YYYY-MM-plan-slug
```

Example:

```text
2026-07-auth-ui-foundation
```

Required files:

```text
PLAN.md
TASKS.md
DECISIONS.md
ACCEPTANCE.md
```

A plan should define its branch name in `PLAN.md`:

```text
Branch: auth-ui-foundation
```

All phases and tasks under that plan should be committed to that branch unless the plan explicitly says otherwise.

Archived or completed plans may move under:

```text
.agent/plans/archive/
```
